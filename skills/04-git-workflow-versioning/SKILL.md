## Branch model (simple)
- `main`: siempre deployable
- `feat/<scope>-<short>`
- `fix/<scope>-<short>`
- `chore/<scope>-<short>`

## Conventional Commits (obligatorio)
- `feat: ...`
- `fix: ...`
- `chore: ...`
- `docs: ...`
- `test: ...`
- `refactor: ...`

## PR rules
- No merge sin PR.
- PR debe incluir: Qué/Por qué, Cómo probar, Screenshots, Riesgos.
- Usar PR template.
